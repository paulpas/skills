## Edge Cases & Failure Modes
- **API endpoint changes**: If skill-router endpoints change, mark Phase 3 as BLOCKED and update curl examples
- **Skill name changes**: If skills are renamed, update all references before Phase 3 validation
- **Missing critical content**: If consolidation reveals missing documentation, pause Phase 2 and create placeholder in appropriate file
- **Validation failures**: If curl commands fail, verify skill-router is running and endpoints match documentation
- **Broken links after symlink removal**: If links break after FAQ.md removal, verify symlink targets and update .gitignore to exclude old files from version control

## When NOT to Use
- Don't consolidate if skill-router API is unstable (expected changes soon)
- Don't remove old files until new documentation has been reviewed and validated
- Don't merge content without checking for breaking changes to user workflows
- Don't proceed without testing API examples against live endpoints
